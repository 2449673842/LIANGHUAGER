# 后续路线图

## 阶段 1：状态持久化

目标：让当前新增的交互不会因为后端重启丢失。

- 自选列表持久化。
- 监控状态持久化。
- 执行账户持久化。
- 执行队列持久化。
- 策略导入记录持久化。

建议实现方式：先用本地 JSON 或 SQLite，后续再升级数据库。

## 阶段 2：真实回测接入

目标：把 `/api/backtests/run` 从本地模拟改成真实回测引擎入口。

- 读取旧 `quant_platform` 的策略、数据和回测参数结构。
- 统一前端调参字段。
- 返回真实指标、交易记录、权益曲线和买卖点。
- 在 K 线上显示买卖点标记。

## 阶段 3：OKX 账户和确认流

目标：让模拟/实盘页具备真实交易前置能力。

- 查询 OKX 账户资产。
- 查询持仓和订单。
- 创建人工确认任务。
- 确认后转为真实下单或模拟下单。
- 保留自动执行开关，但默认关闭。

## 阶段 4：通知和 TradingView 信号

目标：让外部信号能进入策略和执行系统。

- SMTP 邮件通知。
- 企业微信 Webhook。
- TradingView 邮件/Webhook 信号解析。
- 信号进入待确认队列。
- 信号和 K 线买卖点联动。

## 阶段 5：前端质量整理

目标：让工作台更稳定、更可维护。

- 拆分 ECharts bundle。
- 清理生成的伴生 `.js` 文件或明确构建策略。
- 统一按钮、表格、面板和滚动区域尺寸。
- 增加接口错误提示和空状态。
- 增加页面级手动刷新按钮和刷新频率显示。
